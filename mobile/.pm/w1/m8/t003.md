## Objective

Add the legs list to the multi-leg screen: each leg is an account (picked via the existing account picker) plus a signed amount entered inline; the user can append legs with "+ Add leg" and remove any down to a minimum of two. The screen starts with two legs seeded from ledger-meta defaults, mirroring quick add.

## Context

- Account picker: `/(app)/account-picker` (`src/screens/account-picker-screen/`) already lists all account roots, tabbed by prefix; select callbacks go through the `SelectedAssets`/`SelectedExpenses` setter pattern in `src/common/globalFnFactory` — set the fn (closing over the leg index) right before `router.push`.
- Defaults: `useLedgerMeta` (`src/screens/add-transaction-screen/hooks/use-ledger-meta.ts`) — seed leg 1 from the first asset (negative), leg 2 from the first expense (positive), like `quick-add-accounts-selector.tsx`.
- Amount entry: inline `TextInput` with `keyboardType="decimal-pad"` plus a sign toggle (no keypad screen).
- Keep pure leg-state helpers (add/remove/update, parsing amounts to integer cents) in a separate `multi-leg-utils.ts` — the jest-lite runner resolves `@/` only for type imports, so test-reachable value modules must be importable relatively (see t008).

## Steps

1. Define the leg model (`account`, `amount` as string input + parsed cents) and pure helpers in `src/screens/add-transaction-screen/multi-leg-utils.ts`.
2. Render leg rows in a card matching the screen's existing rows: account label (opens account picker with a per-leg callback), amount input right-aligned with the operating-currency symbol, ✕ remove affordance (hidden/disabled at 2 legs).
3. "+ Add leg" row appends an empty leg.
4. Seed initial two legs from `useLedgerMeta` defaults; skeleton/loading background per the loading-state conventions while meta loads.
