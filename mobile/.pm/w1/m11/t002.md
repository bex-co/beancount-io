## Objective

When the user sets a payee in the add flow, fetch that payee's historical accounts and auto-apply the top match to the "to" (expense) account — only if the user hasn't already picked one manually this session. Suggestions must never block or slow the flow.

## Context

- Payee lands via `SelectedPayee` (`src/common/globalFnFactory`) in `src/screens/add-transaction-screen/add-transaction-next-screen.tsx` (`setPayee`).
- Hook home: `src/screens/add-transaction-screen/hooks/use-payee-account-suggestions.ts` (new) — lazy query on payee change, cancel/ignore stale responses.
- Track "user picked manually" so a suggestion never overwrites an explicit choice.

## Steps

1. New hook: given `(ledgerId, payee)`, lazily fetch `getLedgerPayeeAccounts`; expose `{ topMatch, alternatives, loading }`; keep pure ranking/guard logic in a relatively-importable helper for tests.
2. Wire into `add-transaction-next-screen.tsx`: on payee set, auto-apply `topMatch` to the expenses account unless manually chosen; analytics `suggestion_auto_applied`.
3. Failure/slow path: no spinner in the way, no error toast — the flow degrades to today's behavior silently (log to Sentry only).
