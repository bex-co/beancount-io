## Objective

Tapping a category row in the Spending or Income breakdown opens the transactions behind that number — the account-detail screen from w1/m2/t005 driven by `getLedgerAccountJournal` with `with_children: true`, scoped to the report's time range.

## Context

- Target screen: `src/screens/account-detail-screen/` + route `app/(app)/account-detail.tsx` (w1/m2/t005) — extend its params with an optional `time` range instead of building a new screen.
- Category rows carry full account names (e.g. `Expenses:Food`) from t003/t004; `with_children: true` aggregates sub-accounts.

## Steps

1. Extend the account-detail route/screen to accept an optional `time` param and pass it to `use-account-journal` (and hide/adapt the balance chart when a range filter is active if the chart would mislead).
2. Make breakdown rows in the flow report tappable → `router.push` to account-detail with account + current range.
3. Analytics event for report drill-down taps.
