## Objective

Real beancount syntax support in the CM6 editor — vendored from fava, beancount's official web UI — with light and dark editor themes built from the app's theme tokens.

## Context

- fava ships complete, MIT-licensed CM6 beancount modules (`fava/frontend/src/codemirror`): tree-sitter grammar (`tree-sitter-beancount.wasm` via `web-tree-sitter`) plus highlight/indent/fold/autocomplete. Vendor the needed modules into the DOM component bundle with license attribution. Do **not** port editor code from the private `web-beancount` dashboard into this public repo.
- **In-task fallback** if wasm loading inside the DOM-component bundle fights the bundler: a CM6 `StreamLanguage` regex tokenizer covering the same categories (dates, accounts, amounts/commodities, strings, comments, directives, tags/links) — a pure function, unit-testable under jest-lite (relative imports for value modules).
- Themes: two CM6 themes (editor bg, text, gutter, selection, per-token colors) built from `useTheme().colorTheme` tokens passed as props from the native wrapper — the DOM side can't call the hook; never hard-code colors; never compare `themeVar` to `"dark"` directly.
- Stretch (optional, skip if it inflates the task): wire fava's beancount autocomplete with real account names from the existing `getLedgerPayeeAccounts`/`accountHierarchy` data passed as a prop — typing `Expenses:Food:…` on a phone is exactly where completion pays off.

## Steps

1. Vendor fava's beancount language modules (or implement the StreamLanguage fallback) inside `src/components/code-editor/beancount-language/`.
2. Build both themes from token props; wire theme switching through the native wrapper.
3. Verify highlighting on a real ledger file in light **and** dark; keep any hand-written tokenizer pure for t011.
