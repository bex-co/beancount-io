## Objective

Add GraphQL documents for the ledger file API so the native browser/editor can list, read, and write files, then regenerate types.

## Context

- The mobile schema already exposes everything needed (`src/generated-graphql/schema.graphql`): `getLedgerDirContent(dirPath, ledgerId): [LedgerFileContent!]!`, `getLedgerFile(ledgerId, path): LedgerFileContent`, `updateLedgerFile(content, ledgerId, message, path, sha): LedgerFileContent!`.
- `LedgerFileContent` carries `content` (base64 when `encoding === "base64"`), `sha` (optimistic-concurrency token for `updateLedgerFile`), `size`, `type` (file vs dir).
- Documents live in `src/common/graphql/queries/*.graphql` (see `getLedgerErrors.graphql` for style); codegen config in `codegen.ts` picks up `src/common/graphql/**/*.graphql`.

## Steps

1. Add `getLedgerDirContent.graphql`, `getLedgerFile.graphql`, `updateLedgerFile.graphql` selecting `path`, `name` (if present), `type`, `size`, `sha`, `content`, `encoding` as each op needs.
2. Run `yarn codegen`; commit only the regenerated output (never hand-edit `src/generated-graphql/`).
3. `yarn typecheck` green.

## Files

- `src/common/graphql/queries/getLedgerDirContent.graphql` (new)
- `src/common/graphql/queries/getLedgerFile.graphql` (new)
- `src/common/graphql/queries/updateLedgerFile.graphql` (new)
- `src/generated-graphql/` (regenerated)

## Acceptance criteria

- [ ] `useGetLedgerDirContentQuery`, `useGetLedgerFileQuery` (or lazy variants), and `useUpdateLedgerFileMutation` hooks exist in generated output; `yarn typecheck` green.

## Out of scope

- `createLedgerFile` / `deleteLedgerFile` / `renameLedgerFile` (file management is a follow-up; this milestone is browse + edit).
