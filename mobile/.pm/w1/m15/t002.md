## Objective

Replace the Ledger tab's default webview with a native file list: the ledger's files and directories via `getLedgerDirContent`, navigable into subdirectories, with files tappable to open the editor (t005).

## Context

- `src/screens/ledger-screen/ledger-screen.tsx` currently renders `DashboardWebview` pointed at `getEndpoint("ledger/editor/?ledgerId=…")`. Keep `LedgerGuard`, the ledger drawer, and `progress-bar.tsx` behavior; swap only the content area.
- `getLedgerDirContent(dirPath, ledgerId)` returns one directory level; drive navigation with a `dirPath` state + breadcrumb (or push-per-directory), root = ledger root.
- Skeleton first-load per convention: `LoadingTile` rows mirroring the loaded row layout (fixed `lineHeight`s, varied widths), background color set; pull-to-refresh keeps content visible (`refreshing` tracks the pull only).

## Steps

1. Build a `FlatList` of `LedgerFileContent` rows: directory rows (icon + chevron) descend; file rows show name + size, tap → editor route with `{ path }`.
2. Sort directories first, then files, alphabetically; show an honest empty state for an empty directory.
3. Wire breadcrumb/back navigation for `dirPath`; keep drawer + guard + header intact.
4. Remove the webview render path for the default (no `fileUrl`) case only — the `fileUrl` deep-link path stays until t008.

## Acceptance criteria

- [ ] Ledger tab lists real files/dirs for the active ledger; subdirectory navigation and back work; skeleton on first load in light and dark; tapping a file navigates with its `path` param.

## Out of scope
