## Objective

Show the ledger's validation errors for the currently open file inside the editor, with tap-to-jump to the offending line, and refresh them after save so the edit→validate loop closes on mobile.

## Context

- m14 already ships the plumbing: `src/common/hooks/use-ledger-errors.ts` wraps `getLedgerErrors` (errors carry `filename` + `lineno` + message). Filter to the open file's `path` (mind absolute-vs-relative filename forms — normalize by suffix match).
- Display: a compact, dismissible error strip between header and editor (count + first message; expand to list). Errors are advisory — never block editing or saving.
- Jump-to-line: use the `scrollToLine(lineno)` command t003 exposed on `CodeEditor` (a CM6 dispatch with `scrollIntoView` — precise, no line-height math); also honor the `line` route param on mount (used by the notifications deep link in t008).
- Optional bonus (cheap with CM6): also feed the errors into `@codemirror/lint` as diagnostics so the gutter/underline marks the offending lines in-editor.
- Refetch errors after a successful save so the strip reflects the just-saved content.

## Steps

1. Filter `useLedgerErrors` output to the open file; render the strip with theme tokens (error tones in light and dark).
2. Wire strip taps and the `line` param to `CodeEditor`'s `scrollToLine`.
3. Refetch on save success (t005's mutation `onCompleted`).

## Files

- `src/screens/ledger-file-editor-screen/` (error strip + wiring)
- `src/components/code-editor/` (lint-diagnostics wiring if taken)
- `src/translations/en.ts` (strings)

## Acceptance criteria

- [ ] A file with a known error shows the strip with message + line; tapping scrolls to that line; saving a fix clears the strip after refetch; opening with a `line` param lands on that line.

## Out of scope

- Inline squiggles/gutter markers (no gutter in v1); errors for other files (bell/notifications already covers ledger-wide).
