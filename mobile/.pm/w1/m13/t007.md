## Objective

Standing test gate: meaningful unit tests for the behavior this milestone shipped, passing under `yarn test:unit`. Focus on the pure matcher — the piece where regressions (false positives) would be user-visible and embarrassing.

## Context

- Tests live in `src/__tests__/` (e.g. `src/__tests__/brand-avatar.test.ts`), run by the jest-lite runner in `scripts/`.
- jest-lite constraints: **relative imports** for value modules (no `@/` alias), and only `toBe`/`toEqual`/`toBeCloseTo`/`toBeTruthy`/`toBeFalsy`/`toThrow` (+ `.not`) matchers — no component rendering, so test the pure functions.

## Steps

1. Matcher hits: representative real payee strings per category ("Starbucks Store #123", "UBER \*EATS", "Amazon.com\*1A2B3C", "Netflix.com").
2. Matcher misses / false-positive guards: "Amazing Coffee" (not Amazon), "Starbase LLC" (not Starbucks), empty string, whitespace-only, non-Latin payee.
3. Normalization: case, punctuation, extra whitespace all collapse to the same match.
4. `getBrandLogoUri` builds the expected URL for a matched brand.
5. If avatar helpers moved in t002, keep/extend the initials + color determinism tests at their new import path.
