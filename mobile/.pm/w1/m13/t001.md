## Objective

A pure, unit-testable module that maps a free-text payee to a known brand: `matchBrand(payee)` returns `{ name, domain } | null`. Curated map of ~60–100 brands common in consumer ledgers (coffee/food, groceries, transport, subscriptions, retail, travel, finance), each with aliases (e.g. Amazon: `amazon`, `amazon.com`, `amzn`).

## Context

- New module lives with its consumer component: `src/components/transaction-avatar/brands.ts`.
- Payees come from `entry.payee || entry.narration` (`src/screens/journal-screen/journal-entry-item/index.tsx:111`) and `row.title` (`src/screens/account-detail-screen/components/account-entry-row.tsx:89`).
- jest-lite gotcha: this module will be imported by tests via **relative path** (no `@/` alias for value modules) — keep it dependency-free and side-effect-free.

## Steps

1. Define `Brand = { name: string; domain: string; aliases: string[] }` and the curated `BRANDS` list.
2. Implement normalization: lowercase, strip punctuation/diacritics, collapse whitespace.
3. Implement `matchBrand(payee)`: token/word-boundary matching against aliases, longest alias first — substring matching is not acceptable ("Amazing Coffee" must NOT match Amazon; "Subway Sandwiches" SHOULD match Subway).
4. Export `getBrandLogoUri(brand)` building the favicon URL (`https://www.google.com/s2/favicons?domain=<domain>&sz=128`).
