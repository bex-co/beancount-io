## Objective

Add a Monarch-style prominent center "+" button to the tab bar that opens the add-transaction flow from anywhere — it is a button, not a tab: pressing it must never switch the active tab.

## Context

- Tab bar: `app/(app)/(tabs)/_layout.tsx`. expo-router wraps react-navigation bottom tabs, which support a custom `tabBarButton` per screen and `listeners={{ tabPress }}` with `preventDefault()` — **no new dependency**.
- expo-router needs a route file per tab entry: add a stub route (e.g. `app/(app)/(tabs)/add.tsx`) that never renders because the press is intercepted to push `/add-transaction`.
- The capture flow already exists: `app/(app)/add-transaction.tsx` → `add-transaction-next.tsx` (+ payee/narration/account-picker routes).

## Steps

1. Add the stub tab route and register it between Accounts and Journal so the button sits center.
2. Implement the custom `tabBarButton`: raised circular "+" styled with theme tokens (light and dark), adequate hit target, safe-area aware above the home indicator.
3. Intercept the press to push `/add-transaction`; verify the previously active tab stays selected afterwards.

## Files

- `app/(app)/(tabs)/_layout.tsx`
- `app/(app)/(tabs)/add.tsx` (new stub route)

## Acceptance criteria

- [ ] From every tab, tapping "+" opens the add-transaction flow; dismissing it returns to the same tab that was active.
- [ ] The button renders correctly in light and dark with no hard-coded colors.

## Out of scope

- Changes to the add-transaction flow itself.
- AI/natural-language capture (future pillar-2 work).
