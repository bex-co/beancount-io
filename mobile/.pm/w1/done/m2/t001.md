## Objective

Create operation files for three server queries the app does not use yet — `getLedgerBalanceSheet`, `getLedgerAccountJournal`, `getLedgerAccountReport` — regenerate the typed layer with `yarn codegen`, and wrap each in a thin Apollo hook so the Accounts tab and account-detail screen consume typed data.

## Context

- Existing operation files show the pattern: `src/common/graphql/queries/accountHierarchy.graphql`, `homeCharts.graphql`.
- Existing hook pattern: `src/screens/home-screen/hooks/use-account-hierarchy.ts`, `use-home-charts.ts` (ledger id via `useSession` / ledger guard).
- Schema shapes (see `src/generated-graphql/schema.graphql`): `BalanceSheetData` (netWorthData, assetsData, liabilitiesData, hierarchies), `AccountJournalResponse` (items with entry/change/balance, total for pagination), `AccountReport` (linechartData, intervalTotalsData).
- No new npm dependency needed.

## Steps

1. Write `balanceSheet.graphql` selecting `netWorthData`, `assetsData`, `liabilitiesData`, and the assets/liabilities hierarchy trees, parameterized by `ledgerId` and `time`.
2. Write `accountJournal.graphql` over `getLedgerAccountJournal` with `AccountJournalQueryInput` (account, limit, offset, with_children).
3. Write `accountReport.graphql` over `getLedgerAccountReport` (accountName, interval, time).
4. Run `yarn codegen`; never hand-edit `src/generated-graphql/`.
5. Add `use-balance-sheet.ts`, `use-account-journal.ts`, `use-account-report.ts` under `src/screens/accounts-screen/hooks/`.
