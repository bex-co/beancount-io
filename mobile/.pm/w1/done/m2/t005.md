## Objective

Tapping an account row opens a detail screen: the account's balance history as a line chart (`getLedgerAccountReport`) above a paginated list of its journal entries with running balances (`getLedgerAccountJournal`) — Monarch's account drill-down.

## Context

- Hooks from t001: `use-account-report` (linechartData), `use-account-journal` (items: entry/change/balance, `total` + limit/offset for pagination, `with_children: true` for parent accounts).
- Route pattern for non-tab screens: `app/(app)/account-picker.tsx` etc.; pass the account name via router params.
- Entry rendering: reuse row styling/utilities from `src/screens/journal-screen/` where sensible.
- Chart: `src/common/d3/line-chart-d3.tsx`; range via `src/components/time-range-pills/`.

## Steps

1. Create `src/screens/account-detail-screen/` (screen + components) and route `app/(app)/account-detail.tsx` reading an `account` param.
2. Render header (account name, current balance), balance-history chart, and entry list with change + running balance per row.
3. Paginate with `offset`/`limit` (infinite scroll or "load more"), guarding against refetch loops.
4. Wire `onPressAccount` from t004 to `router.push` with the account param.

## Files

- `app/(app)/account-detail.tsx` (new)
- `src/screens/account-detail-screen/account-detail-screen.tsx` (new) + `components/`, `index.ts`

- [ ] Tapping any account row on the Accounts tab opens its detail screen

- [ ] "Load more" fetches the next page and appends without duplicates

## Out of scope

- Category drill-down from Reports (w1/m3/t007 reuses this screen).
