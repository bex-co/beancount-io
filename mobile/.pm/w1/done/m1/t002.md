## Objective

Build a reusable horizontally-swipeable carousel with a page-dot indicator (the 3-dot control in the Monarch reference), so the account-charts card (t003) can page between net worth, spending-by-category, and liabilities. Reuse the already-installed `react-native-pager-view` — no new dependency.

## Context

- `react-native-pager-view` (v8.0.2) is already a dependency; `src/components/tabs/index.tsx` shows the established `<PagerView>` usage (initial index, `onPageSelected`, bidirectional sync) — learn from it, but that component renders a tab bar with an underline, **not** the dots we need.
- Theme/i18n: `useTheme().colorTheme`, `useThemeStyle`; active/inactive dot colors from theme tokens (e.g. `theme.primary` vs a muted gray token).

## Steps

1. Create `src/components/paged-carousel/index.tsx`: props `pages: React.ReactNode[]` (or `{ key, render }[]`), optional `onPageChange?`, optional `initialIndex`. Wrap `<PagerView>`, track the selected page via `onPageSelected`, and render a row of dots below (active dot uses `theme.primary`, inactive a muted token).
2. Size the pager to its content height (fixed card height via prop, since PagerView needs a bounded height) — accept a `height` prop consumed by the account-charts card.
3. Ensure the dots are tappable-optional but at minimum reflect swipe position; keep accessibility labels localized where a label is shown.

## Files

- `src/components/paged-carousel/index.tsx` (new)
- `src/translations/en.ts` (only if a visible/a11y string is added)

## Acceptance criteria

- [ ] Swiping horizontally advances pages and the active dot updates to match.
- [ ] Dot colors come from theme tokens and read correctly in light and dark.
- [ ] Component is self-contained and reusable (no home-screen-specific coupling).

## Out of scope

- Wiring the three specific chart pages (that's t003). This task ships the generic carousel only.
