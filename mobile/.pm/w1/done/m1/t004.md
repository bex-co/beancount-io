## Objective

Add a "Recent transactions" card to the dashboard (Monarch's "Transactions · Most recent" list) showing the latest few entries with payee/narration + signed amount, plus a "see all →" affordance that navigates to the journal tab. Gives users an at-a-glance view of recent activity without leaving home.

## Context

- Card chrome from t001 (`src/components/dashboard-card/` with `title` + `onSeeAll`).
- Existing journal infra to reuse (do not reinvent the row):
  - `src/screens/journal-screen/journal-screen.tsx` — `useGetLedgerJournalQuery` fetch pattern (pagination via `first`/`after`).
  - `src/screens/journal-screen/journal-entry-item/index.tsx` — `JournalEntryItem` row (date · flag · description + amount); `journal-item-description.tsx`.
  - `src/common/graphql/queries/journalEntries.graphql` — fields `date, payee, narration, netAmount, primaryAccount, postings…` (already generated).
- Navigation: the journal tab route is `app/(app)/(tabs)/journal.tsx`; use `expo-router` `useRouter().navigate(...)` as `home-screen.tsx` already does for `/add-transaction`.

## Steps

1. Build `src/screens/home-screen/components/recent-transactions-card.tsx`: fetch the latest ~4–5 entries with `useGetLedgerJournalQuery` (`first: 5`, newest-first sort), render each with the existing `JournalEntryItem` (or a compact variant if the full row is too tall for a card).
2. Wrap in `DashboardCard` with `title={t("recentTransactions")}` and `onSeeAll` → navigate to the journal tab.
3. Handle loading (skeleton with background color) and empty state (reuse/mirror `journal-empty-state.tsx` messaging).
4. Mount the card in `home-screen.tsx` below the account-charts card; include it in `onRefresh` if it exposes a refetch.
5. Add `recentTransactions` (+ any empty-state string) to `src/translations/en.ts`.

## Acceptance criteria

- [ ] Card shows the latest few transactions with payee/narration and signed amount, matching how the journal screen renders them.
- [ ] "See all →" navigates to the journal tab.
- [ ] Loading skeleton and empty state both render with correct backgrounds in light and dark.

## Out of scope

- Editing/opening a transaction detail from the card (tapping a row can defer to existing journal behavior or be a no-op glimpse); month-over-month spending (t005).
