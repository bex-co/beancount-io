## Objective

Add the editor foundation: a `CodeEditor` component built as an Expo DOM component (`'use dom'`) bundling CodeMirror 6 locally, with keyboard-first configuration — and an on-device keyboard validation gate before the milestone builds on it.

## Context

- Owner decision 2026-07-14 (m15 README Source): beancount is code — a real code editor (CM6) over the live-markdown TextInput-decoration approach.
- ⚠ **NEW DEPENDENCIES — ask the user before adding (repo rule):** `codemirror` (+ its `@codemirror/*` core packages — all pure JS, bundled into the DOM component, no native linking) and `react-native-keyboard-controller` (native; used by t006's accessory bar). t004 may add `web-tree-sitter` for fava's grammar. `react-native-webview` 13.16.1 is already installed (DOM components render through it). Install via `yarn add` from inside `mobile/`; never hand-edit `yarn.lock`.
- Expo DOM components: a React web component compiled into a locally-bundled webview with typed props and function-prop RPC — Expo's docs list code editors as a target use case. Needs dev client/EAS (already the case), not Expo Go. Unlike today's ledger webview, nothing is fetched from the network.
- Keyboard-first config lives on the CM6 side via `EditorView.contentAttributes`: `autocapitalize="off"`, `autocorrect="off"`, `spellcheck="false"` (also neutralizes iOS smart quotes — beancount narrations need straight `"`). On the native side, set `hideKeyboardAccessoryView` on the underlying webview (iOS) so t006's bar is the only bar.
- Split: `src/components/code-editor/index.tsx` (native wrapper: props in, theme tokens, imperative `insertAtCursor`/`scrollToLine` plumbing) + `src/components/code-editor/code-editor.dom.tsx` (`'use dom'`: CM6 `EditorView`, monospace, `lineNumbers()`, `initialContent` prop, `onChangeText` function prop; expose insert/scroll commands via the DOM component's imperative handle or a versioned command prop).

## Steps

1. Get user approval for the dependencies, then `yarn add` them.
2. Build the DOM component + native wrapper as above.
3. **Keyboard validation gate (on-device, both platforms):** type a realistic transaction fast and verify the known CM6 mobile quirks are absent or mitigated — iOS autocapitalize at line start (codemirror/dev#433; the `contentAttributes` should suppress it — verify), swipe-typing stray spaces, Android selection/delete edge cases in RN WebView (codemirror/dev#645), cursor scrolls into view when the keyboard opens. Also load a 2–5k-line file and type mid-file (CM6 virtualizes — confirm smooth). If the keyboard feel fails, stop and flag to the user; fallbacks are `@expensify/react-native-live-markdown` (native TextInput decoration, color-only) or the v2 native module in inbox note `w1/008`.
