## Objective

Add the three read-only operation documents this milestone consumes and regenerate typed hooks, so the header badge, notifications screen, and commit detail all build on generated types.

## Context

- Operation documents live in `src/common/graphql/queries/*.graphql` (see `getLedgerPayeeAccounts.graphql` from m11 as the pattern).
- Schema (already in `src/generated-graphql/schema.graphql`):
  - `getLedgerErrors(ledgerId: String!): [BeancountError!]!` — `filename`, `lineno`, `message`.
  - `listCommits(ledgerId: String!, branch: String! = "main", page: Int! = 1, limit: Int! = 30): [CommitListItem!]!` — `sha`, `shortSha`, `message`, `author`.
  - `getCommitDetails(ledgerId: String!, sha: String!): CommitDetails!` — `message`, `author`, `stats { additions deletions total }`, `files { … }`, `diff`.

## Steps

1. Create `getLedgerErrors.graphql`, `listCommits.graphql`, `getCommitDetails.graphql` under `src/common/graphql/queries/`, selecting the fields above.
2. Run `yarn codegen`; commit the regenerated output as-is (never hand-edit `src/generated-graphql/`).
3. `yarn typecheck` green.

## Files

- `src/common/graphql/queries/getLedgerErrors.graphql` (new)
- `src/common/graphql/queries/listCommits.graphql` (new)
- `src/common/graphql/queries/getCommitDetails.graphql` (new)
- `src/generated-graphql/` (regenerated via `yarn codegen` only)
