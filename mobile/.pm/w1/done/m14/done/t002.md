## Objective

Add a bell button immediately to the right of the hamburger in the shared tab header. It shows a small count badge when `getLedgerErrors` returns errors, and tapping it navigates to the new notifications screen (t003) with an analytics event.

## Context

- The hamburger is `LedgerDrawerButton` inside `LedgerDrawerHeader` (`src/components/ledger-drawer/ledger-drawer-header.tsx`); the left cluster is the `navLeft` view (width 64, `flex-start`) — the bell slots in there, keeping the centered title balanced against `navRight`.
- Badge count from a small `useLedgerErrors()` hook wrapping the generated `getLedgerErrors` query (current ledger id per existing hooks' pattern, e.g. `src/screens/add-transaction-screen/hooks/`).
- Icons via `@expo/vector-icons` `Ionicons` (e.g. `notifications-outline`), colors via `useTheme()` tokens only.

## Steps

1. Create `useLedgerErrors()` in `src/common/hooks/` returning `{ errors, count, loading }`; errors failing to load ⇒ count 0 (never block the header).
2. Add a `NotificationsBellButton` next to `LedgerDrawerButton` in `LedgerDrawerHeader`; render a badge (theme danger/primary token background) when count > 0.
3. On press: `analytics.track("tap_notifications_bell", { errorCount })` then `router.push` to the notifications route (t003 creates it; land this task with the route name agreed).
4. Verify the title stays visually centered on all five tabs, light and dark.

## Files

- `src/components/ledger-drawer/ledger-drawer-header.tsx`
- `src/common/hooks/use-ledger-errors.ts` (new)

## Acceptance criteria

- [ ] Bell renders right of the hamburger on every tab header; badge appears only when the ledger has errors.
- [ ] Query failure or loading shows no badge and never blocks the header.
- [ ] Tap tracked in analytics and navigates to the notifications route.

## Out of scope

- The notification list itself (t003); push notifications of any kind.
