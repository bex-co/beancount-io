## Objective

The bell opens a notification list with two sections: **Errors** — each ledger error with its message and `filename:lineno` context — and **Recent changes** — the latest commits to the ledger (message, author, short SHA). Errors sit on top so a broken ledger is impossible to miss; a clean ledger shows an honest empty state for the errors section with changes below.

## Context

- New route `app/(app)/notifications.tsx` mounting `src/screens/notifications-screen/` (pattern: `app/(app)/settings.tsx` → `src/screens/setting/`).
- Data: `useLedgerErrors()` from t002 and the generated `listCommits` hook (t001), default branch/page/limit.
- First load renders a `LoadingTile` skeleton mirroring the list layout (varied widths, fixed line heights); pull-to-refresh keeps content visible per convention.
- Tapping a change row pushes the commit detail route (t004) with the commit `sha` as a param.

## Steps

1. Create the route + screen with `SafeAreaView`, header title, and analytics on mount (`view_notifications`).
2. Errors section: red/danger-token accent, message + `filename:lineno` (filename may be null — render message-only gracefully).
3. Recent-changes section: commit message (1–2 lines), author name, `shortSha` in monospace; tap → commit detail.
4. Empty states: no errors ⇒ localized "ledger is healthy" line; no commits ⇒ localized empty line.
5. Skeleton first-load state; both sections in light **and** dark.

## Files

- `app/(app)/notifications.tsx` (new)
- `src/screens/notifications-screen/` (new)
- `src/translations/` English base (section titles, empty states)

## Acceptance criteria

- [ ] With a broken ledger, errors appear at the top with file/line context; with a clean ledger, the healthy empty state shows and changes render below.
- [ ] Tapping a change opens commit detail with the right `sha`.
- [ ] Skeleton on first load; pull-to-refresh refetches both queries without blanking content.

## Out of scope

- Commit diff rendering (t004); pagination beyond the default `listCommits` page; any non-ledger notification sources.
