## Objective

Tapping a change in the notification list opens the full picture of what that write did to the ledger: commit message and author, per-file +/− stats, and the unified diff rendered in monospace with added/removed lines tinted via theme tokens — the audit trail behind every mobile write.

## Context

- New route `app/(app)/commit-detail.tsx` taking a `sha` param, mounting `src/screens/commit-detail-screen/`.
- Data: generated `getCommitDetails(ledgerId, sha)` hook (t001) — `message`, `author`, `stats`, `files` (per-file additions/deletions), `diff: String` (may be null/large).
- Diff rendering: split `diff` into lines; tint `+` lines with the theme success/green token and `-` lines with the danger/red token (background tint, monospace font). No syntax-highlighting dependency — plain text only (no new dependencies).

## Steps

1. Create route + screen: header (short SHA as title), `SafeAreaView`, analytics on mount (`view_commit_detail`).
2. Summary block: message, author, total +/− stats; files list with per-file additions/deletions.
3. Diff block: line-by-line monospace rendering with theme-token tinting; horizontal scroll for long lines; null/empty diff renders a localized "no diff available" line.
4. Skeleton first-load state; verify a large diff scrolls smoothly (FlatList over lines if needed).

## Files

- `app/(app)/commit-detail.tsx` (new)
- `src/screens/commit-detail-screen/` (new)
- `src/translations/` English base (labels, empty diff)

## Acceptance criteria

- [ ] From a notification change row, the detail shows message, author, stats, files, and a tinted monospace diff.
- [ ] Null/empty diff and query errors render honest localized states, not blanks.
- [ ] No hard-coded colors; correct in light **and** dark.

## Out of scope

- Reverting/checking out commits; commit pagination or browsing beyond notification taps; syntax highlighting.
