## Objective

An "Edit" action on transaction detail opens a screen with the entry's exact beancount source text in a monospace multiline editor; saving writes it back via `updateLedgerEntrySourceSlice`. Raw-text editing is the point: it is lossless by construction (metadata, tags, costs, comments all survive), which a structured editor could not guarantee.

## Context

- Source text + `sha256sum` already available on `src/screens/transaction-detail-screen/transaction-detail-screen.tsx` via `getLedgerEntryContext`.
- New route in `app/(app)/` (pattern: `transaction-detail.tsx`); screen colocated under `src/screens/transaction-detail-screen/`.
- Server validates the slice — invalid beancount comes back as a mutation error; surface it inline/toast, never save silently.
- Keyboard handling matters: multiline monospace `TextInput`, `SafeAreaView`, keyboard-avoiding behavior.

## Steps

1. Route `app/(app)/edit-transaction.tsx` + `src/screens/edit-transaction-screen/edit-transaction-screen.tsx`; payload passed via reactive var; `usePageView("edit_transaction")`.
2. Monospace editor prefilled with `slice`; Save in the header (disabled while saving); Cancel discards.
3. Save via `updateLedgerEntrySourceSlice` with `sha256sum`; success toast → back; server validation errors and stale-sha conflicts shown honestly with the text preserved for retry.
