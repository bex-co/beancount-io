## Objective

Meaningful unit tests under `src/__tests__/` for the typography system's real invariants — the rules other code depends on, not snapshot trivia.

## Context

- Runner: `yarn test:unit` (jest-lite in `scripts/`). The runner resolves `@/` only for **type** imports — value imports in test-reachable modules must be relative, so import `typography.ts` relatively from the test.
- Invariants worth asserting: `amountStyle` carries `fontVariant: ["tabular-nums"]`; the size scale exposes no step below its documented minimum and mono minimum is ≥13; `fontWeights` exposes exactly regular/medium; `fonts.mono` resolves to a non-empty family on both `Platform.OS` branches (mock `Platform`).
- Anti-gaming bar: each test must fail if the corresponding rule regresses (e.g. someone deletes `tabular-nums` or adds a `light` weight) — no tautologies.

## Steps

1. Add `src/__tests__/typography.test.ts` covering the invariants above.
2. Confirm each test fails when its invariant is deliberately broken, then passes on the real module.
3. `yarn test:unit` green.
