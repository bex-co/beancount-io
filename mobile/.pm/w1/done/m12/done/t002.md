## Objective

Embed JetBrains Mono (Regular + Medium) natively so ledger text uses one legible mono on both platforms, and remove the dead SpaceMono asset.

## Context

- `app.json` already carries an `expo-font` config plugin entry pointing at `./src/assets/fonts/SpaceMono-Regular.ttf` — but `SpaceMono` is referenced nowhere in `src/` or `app/`; it's the Expo template leftover shipping dead weight in the binary.
- **No new npm dependency** — `expo-font ~57.0.0` is already installed and the plugin is already configured; this task only swaps vendored `.ttf` assets. JetBrains Mono is OFL-licensed; vendor the license file alongside.
- JetBrains Mono over Space Mono for legibility: taller x-height, dotted zero, disambiguated `1/l/I` and `0/O` — this is ledger text full of digits.
- Native config change: the `ios/` project must be regenerated (`npx expo prebuild`) / dev client rebuilt for the embedded font to load.

## Steps

1. Vendor `JetBrainsMono-Regular.ttf` + `JetBrainsMono-Medium.ttf` + `OFL.txt` into `src/assets/fonts/`; delete `SpaceMono-Regular.ttf`.
2. Update the `expo-font` plugin entry in `app.json` to the two JetBrains Mono files.
3. Point `fonts.mono` in `src/common/theme/typography.ts` at the embedded family names (keep the Menlo/monospace fallback for environments where the embed is absent, e.g. Expo Go).
4. Rebuild the dev client and confirm the font renders on the iOS simulator and Android.

## Files

- `src/assets/fonts/` (add JetBrainsMono-Regular.ttf, JetBrainsMono-Medium.ttf, OFL.txt; remove SpaceMono-Regular.ttf)
- `app.json`
- `src/common/theme/typography.ts`

## Acceptance criteria

- [ ] A test `Text` with `fonts.mono` renders JetBrains Mono (dotted zero visible) on iOS **and** Android dev builds.
- [ ] `SpaceMono-Regular.ttf` gone from the repo and from `app.json`.
- [ ] `OFL.txt` vendored next to the fonts.

## Out of scope

- Applying the font to screens (t003); UI (non-mono) custom fonts — deliberately not doing this (system font keeps Dynamic Type + Cyrillic/Farsi coverage).
