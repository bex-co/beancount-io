## Objective

Move every journal-screen mono declaration onto the theme's `fonts.mono` token, fixing the shipped iOS bug where `fontFamily: "monospace"` silently falls back to San Francisco, and scope mono to structured ledger text only.

## Context

- Broken on iOS today (raw `"monospace"` is not a valid iOS font name): `src/screens/journal-screen/journal-entry-item/journal-item-date.tsx:11`, `journal-item-flag.tsx:12`, `journal-item-description.tsx:46`, `src/screens/journal-screen/journal-bottom-sheet/balance-section.tsx:35,41`.
- Correct-but-local: `src/screens/journal-screen/journal-bottom-sheet/sheet-content.tsx:12` defines its own `MONO_FONT = Platform.select({ ios: "Courier New", … })` — delete in favor of the token.
- Legibility rule from the design pass: mono for structured text (dates, flags, amounts, account names) at ≥13; free-form payee/narration in the system font — mono eats ~15–20% more width and truncates narrations sooner.

## Steps

1. Replace the five `fontFamily` declarations with `fonts.mono`; bump any mono size below 13 to 13.
2. In `journal-item-description.tsx`, split rendering: account/amount fragments keep mono, narration text drops to the system font.
3. Delete the local `MONO_FONT` const in `sheet-content.tsx`; use the token.
4. Eyeball the journal list and bottom sheet on iOS + Android, light + dark.
