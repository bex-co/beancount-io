## Objective

Converge ad-hoc `fontSize`/`fontWeight` values on the main tabs onto the theme scale from t001 — legibility floor enforced (no mono below 13, no body text below the scale's minimum), weights limited to regular/medium.

## Context

- Current spread: 151 `fontSize` and 90 `fontWeight` declarations across `src/screens` + `src/components`; most are near-misses of each other (e.g. 13/14/15 used interchangeably for the same role).
- Scope to the high-traffic surfaces: `src/screens/home-screen/`, `src/screens/journal-screen/`, `src/screens/accounts-screen/`, `src/screens/account-detail-screen/`, `src/screens/add-transaction-screen/`, plus shared components they mount (`src/components/dashboard-card`, `account-list`, `time-range-pills`, `split-button`).
- Watch the skeleton convention: real text has fixed `lineHeight`s sized to match `LoadingTile` skeletons — when a `fontSize` changes, resize the paired tile + margins so the view still doesn't shift when data lands (`src/screens/add-transaction-screen/list-item.tsx`, account-picker loading state, text-input-screen suggestions card).

## Steps

1. Map each declaration on the scoped screens to the nearest scale step; flag any that genuinely need a new step rather than forcing it.
2. Replace `fontWeight` literals with the theme's regular/medium constants; demote bold amounts to medium.
3. Re-check paired skeleton `lineHeight`s wherever a size changed.
4. `yarn lint` green.

## Files

- `src/screens/{home-screen,journal-screen,accounts-screen,account-detail-screen,add-transaction-screen}/**`
- `src/components/{dashboard-card,account-list,time-range-pills,split-button}/**`

## Acceptance criteria

- [ ] On the scoped screens, every `fontSize`/`fontWeight` comes from the theme scale (grep shows no numeric literals there, or each remaining literal has a one-line justification).
- [ ] No mono text below 13; no light/bold weights introduced.
- [ ] Skeleton layouts still match loaded layouts pixel-for-pixel (no shift when data lands).
