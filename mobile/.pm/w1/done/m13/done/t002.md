## Objective

One shared `TransactionAvatar` component that renders the brand logo when `matchBrand` hits, and the existing colored-initials circle otherwise — same 40×40, borderRadius-20 geometry in both states so adopting it causes zero layout shift.

## Context

- New component: `src/components/transaction-avatar/index.tsx`; export from `src/components/index.ts` alongside the existing components.
- Reuse `getAvatarInitials` / `getAvatarColor` from `src/screens/journal-screen/utils/journal-utils.ts` for the fallback (move them into the component's module if that removes the cross-screen import; keep re-exports if journal-utils tests depend on them).
- Theme via `useTheme().colorTheme` — no hard-coded colors except the existing `#fff` initials text pattern already used in both rows.

## Steps

1. Props: `{ name: string; size?: number }` (default 40). Call `matchBrand(name)`.
2. Matched: render RN `Image` with `getBrandLogoUri`, on a theme-aware neutral chip (e.g. `theme.black10` background) so transparent/dark logos stay visible in dark mode; slight padding so favicons don't touch the circle edge.
3. Track `onError` in state → fall back to initials rendering (also covers offline). Render initials underneath / as placeholder until `onLoad` so there is never a blank circle while the image fetches.
4. Unmatched: exactly today's initials avatar (`getAvatarColor` background + `getAvatarInitials` text).
