## Objective

Add a Monarch-style menu section to the ledger drawer: below the ledger list, a Settings row (gear icon + label) that closes the drawer and pushes `/settings`; optionally a profile row (user email) doing the same, mirroring Monarch's bottom-of-menu profile entry.

## Context

- Drawer: `src/components/ledger-drawer/ledger-drawer.tsx` — brand header + `Ledgers` section label + `FlatList`; menu section goes after the list (bottom-pinned like Monarch is fine, the list is short).
- Close-then-navigate: reuse `requestClose()`; push after closing so the reveal animation doesn't fight the stack transition.
- i18n: reuse the existing `setting` key ("Setting") or add `settings` to `src/translations/en.ts`; render via `useTranslations()`.
- Email lookup (optional row): `useUserProfile` in `src/screens/setting/hooks/` — skip if it drags in heavy deps; the gear row is the required piece.
- Add `testID="ledger-drawer-settings"` for simulator automation (see expo-mcp driving notes).

## Steps

1. Add a divider + menu section to the drawer with a Settings row (Ionicons `settings-outline`, theme tokens only).
2. On press: `analytics.track("drawer_open_settings", {})`, close the drawer, `router.push("/settings")`.
3. Optional: profile row with the user's email above the Settings row.

## Files

- `src/components/ledger-drawer/ledger-drawer.tsx`
- `src/translations/en.ts` (only if a new key is needed)

## Acceptance criteria

- [ ] Drawer shows the Settings row under the ledger list in light and dark.
- [ ] Tapping it closes the drawer and lands on `/settings`; back returns to the tab you came from with the drawer closed.
- [ ] `yarn lint` and `yarn test:unit` green.
