## Objective

Make the existing Settings screen reachable as a pushed stack screen at `/settings` (sibling of `/account-detail`), with a native back button, so the drawer can link to it and the tab can later be removed.

## Context

- `src/screens/setting/setting-screen.tsx` mounts `About` inside `LedgerGuard`; today it only exists as the `setting` tab.
- `src/screens/setting/account-header.tsx` renders the purple banner with its own `SafeAreaView edges={["top"]}` **and** a `LedgerDrawerButton` — the button calls `useLedgerDrawer()`, which throws outside `LedgerDrawerProvider` (the provider wraps only the tab group in `app/(app)/(tabs)/_layout.tsx`). A pushed `/settings` screen is outside the provider.
- Route pattern to copy: `app/(app)/account-detail.tsx` + default stack header from `app/(app)/_layout.tsx` (`DefaultHeaderLeftBack`).

## Steps

1. Create `app/(app)/settings.tsx` mounting `SettingScreen` (same shape as `account-detail.tsx`).
2. Remove `LedgerDrawerButton` from `account-header.tsx` (pushed screens navigate back, they don't open the drawer — Monarch does the same).
3. Reconcile the double top inset: with the stack header shown, the banner's `SafeAreaView edges={["top"]}` over-pads — either set `edges={[]}` when headerShown, or hide the stack header and keep the banner's inset + a back affordance. Pick one; test on the notch simulator.
4. Verify logout, delete-account, and ledger-selection flows still work when launched from the pushed screen.

## Files

- `app/(app)/settings.tsx` (new)
- `src/screens/setting/account-header.tsx`
- `src/screens/setting/setting-screen.tsx` (if inset/header handling needs a prop)

- [ ] Navigating to `/settings` (e.g. via a temporary `router.push`) shows the full settings screen with a working back button and no double top inset.
- [ ] No `useLedgerDrawer must be used within a LedgerDrawerProvider` crash on the pushed screen.
- [ ] `yarn lint` and `yarn test:unit` green.
