## Objective

Make the transactions list on the account detail screen share the Journal tab's visual language — avatar initials, name/amount typography and coloring, and date section grouping — instead of its current bespoke row, while keeping the running balance visible on each entry.

## Context

- Current: `src/screens/account-detail-screen/account-detail-screen.tsx` renders a flat `FlatList` of `AccountEntryRow` (`src/screens/account-detail-screen/components/account-entry-row.tsx`) — title/date left, signed change + running balance right, hairline separators. Data rows come from `selectAccountJournalRows` (`src/screens/account-detail-screen/selectors/select-account-journal.ts`).
- Target look: Journal (`src/screens/journal-screen/journal-screen.tsx`) — `SectionList` grouped by `JournalDateSectionHeader` with `JournalEntryItem` rows (`src/screens/journal-screen/journal-entry-item/index.tsx`: avatar via `getAvatarColor`/`getAvatarInitials`, name + pending badge, colored amount).
- Reuse precedent: `src/screens/home-screen/components/recent-transactions-card.tsx` already reuses `JournalEntryItem` (commit `a9dd7e9`). Prefer reusing/parameterizing the Journal components over duplicating their styles; if `JournalEntryItem` needs a subtitle/right-secondary slot for the running balance, extend it rather than forking.
- Preserve existing behavior: pagination via `fetchMore` + `onEndReached`, pull-to-refresh, initial-loading/error/empty states, loading-more + no-more footers.

## Steps

1. Decide the reuse shape: render `JournalEntryItem` (extended with a running-balance secondary line) from account-journal rows, grouped by date like Journal, or extract the shared row/section styles if direct reuse doesn't fit `AccountJournalRow`.
2. Add/adjust a selector to group account-journal rows into date sections compatible with `JournalDateSectionHeader`.
3. Swap the account-detail list rendering to the Journal-style rows + section headers; keep the balance chart card and "Transactions" section title as the list header.
4. Remove `AccountEntryRow` if fully superseded (or slim it to a thin wrapper).
5. Verify pagination, refresh, and empty/error/footer states still work; check light and dark.
