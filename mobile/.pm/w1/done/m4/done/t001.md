## Objective

The back button on pushed stack screens (account detail being the visible case) renders with no arrow glyph — only an empty tappable area. Restore a visible back arrow that follows the theme.

## Context

- `app/(app)/_layout.tsx` — `DefaultHeaderLeftBack` renders `HeaderBackButton` from `expo-router/react-navigation` with only `onPress` + `tintColor`, wired as the stack-wide `headerLeft`. The glyph stopped rendering after the Expo 57 / react-navigation dep bump (`694f79a`).
- Affects every screen pushed in the `(app)` stack, e.g. `app/(app)/account-detail.tsx`.
- No new npm dependency expected; if a fix would require one, stop and ask the user first (repo rule).

## Steps

1. Reproduce: push Accounts → any account; confirm the header shows no arrow.
2. Diagnose why `HeaderBackButton` renders no icon post-upgrade (e.g. `displayMode`/`backImage` prop changes, or the custom `headerLeft` suppressing the default icon).
3. Fix — either pass the props the new version needs, or drop the custom `headerLeft` in favor of the stack's native back button with themed `headerTintColor` if that is simpler and behaves identically.
4. Verify the arrow renders and navigates back in light **and** dark themes (tint from theme tokens, no hard-coded colors).
