# Scope v2: true native code-editor module (Runestone + sora-editor)

If m15's color-only highlighting ever needs to become a real code editor (line numbers, grammar-accurate tree-sitter highlighting, search/replace, whole-file scale), the researched route (2026-07-14) is a custom Expo Module wrapping **Runestone** (iOS, Core Text, tree-sitter — pin grammar to ABI 14: `tree-sitter-beancount` v2.3.3 or `--abi=14`) and **sora-editor** (Android, LGPL-2.1, actively maintained; loads `Lencerf/vscode-beancount`'s MIT `beancount.tmLanguage` from assets at runtime).

No such wrapper exists on npm (verified sweep); the only prior art is `workspace-sh/react-native-source-editor` (MIT, dormant, unpublished, 0 stars) — don't depend on it, but it's working scaffold for the Fabric view + SPM config-plugin plumbing (~40% of the work). Design must be uncontrolled (initial-content prop + change/selection events + imperative `setText`); iOS needs `spm_dependency`/config plugin (Runestone is SPM-only) and manual keyboard-inset handling.

Honest estimate: **12–19 person-days** to first shippable plus permanent maintenance of two native deps on different release heartbeats. Both editors are custom text-input implementations whose keyboard behavior is good but historically buggier than the stock TextInput m15 rides on. Only worth it if product requirements outgrow the 7-color-channel approach — not committed.
